Question 01

选择最 SOTA 的 Retriever，
理想情况下来自最新的 GraphRAG 工作

"我们可能有必要选几个最sota的retriever来测试，替换现在的 retriever，ideally retriever应该是来自于现在最sota的那些GraphRAG的work。"

基于对当前 GraphRAG 领域的调研，选定以下三个最具代表性且与癫痫知识图谱结合最紧密的 Retriever， 覆盖神经科学启发设计、图结构问答、以及轻量级高效检索三个不同方向。

🧠
HippoRAG
Neurobiologically Inspired Long-Term Memory for Large Language Models
Guti et al. · NeurIPS 2024
Selected ✓
核心思路
模拟海马体（Hippocampus）的记忆巩固机制。将文档解析为知识三元组 (subject, relation, object)，构建 开放领域知识图谱；检索时结合 PPR（Personalized PageRank）在图上扩散，找到上下文关联的多跳知识节点。
与癫痫的契合点
癫痫知识天然具有图结构：SCN1A → causes → Dravet Syndrome → treated_by → Clobazam。HippoRAG 能跨论文追踪基因-表型-治疗链，是所有 RAG 方法中最适合多跳推理的。
实验状态
已实现（retriever_hipporag_ppr.py）· 当前主要 RAG 系统 kg_rag 即基于此框架
优势
多跳推理能力强；知识持久化（long-term memory）；PPR 检索质量高；与 LLM 解耦（不需重训）
局限
依赖 LLM 抽取三元组（成本高）；构建图需要预处理时间；PPR 计算有一定延迟
🔗
G-Retriever
Retrieval-Augmented Generation for Textual Graph Understanding and Question Answering
He et al. · NeurIPS 2024
Candidate ★
核心思路
专为文本图（Textual Graph）设计的 RAG 框架。将知识图谱中的子图作为检索单元，通过 Prize-Collecting Steiner Tree (PCST) 算法找到最相关子图，再将子图序列化后注入 LLM。
与癫痫的契合点
适合结构化图谱查询：从 OMIM/ClinVar/癫痫指南构建的知识图谱中，直接检索"基因变异→综合征→用药"子图路径。比 HippoRAG 更精确地返回图结构证据。
实验状态
建议下一步实现——适合 T3（Variant→ASM）和 T5（多模态诊断）任务
优势
显式图结构推理；支持问题引导的子图裁剪；hallucination 率低；与结构化医学数据库天然兼容
局限
需要高质量结构化图谱；PCST 算法对大图计算成本较高；不适合纯文本语料
⚡
LightRAG
Simple and Fast Retrieval-Augmented Generation
Edge et al. · EMNLP 2025
Candidate ★
核心思路
双层检索（Low-Level + High-Level）机制：Low-level 检索实体级别的精确事实；High-level 检索主题级别的全局摘要。两层结果合并后注入 LLM，兼顾精度与覆盖。
与癫痫的契合点
癫痫 QA 任务的问题跨度大：既有高精度事实问题（"SCN1A的ACMG分类？"），也有宏观总结问题（"Dravet综合征的治疗进展？"）。LightRAG 的双层机制天然适配 T1（New-Paper QA）。
实验状态
已有代码基础（retriever_lightrag_mix.py）· 建议替换并系统对比
优势
构建速度快（"Simple and Fast"）；双层检索覆盖不同粒度问题；无需大型图谱预处理；易于部署
局限
图结构语义没有 HippoRAG/G-Retriever 丰富；多跳推理能力相对较弱

Question 03

5 大 Task 的完整 Setup 设计、
Evaluation 方法与 Baseline 选取

"我们要完整的给出4-5大类task的setup设计，evaluation方法，等等；然后为跑一个完整的benchmark paper做准备。与此同时，每一类task，我们都要有一组baseline，来对比performance。这些baseline，除了最好的那些闭源的LLM之外，最好还有一些来自于sota的paper里提出的方法。我们要让reviewer觉得，每一个task上我们选取的baselines都是目前最好最有竞争力的了。"
T1 · New-Paper QA ★★★★★ T2 · Patient Explanation ★★★★★ T3 · Variant→ASM ★★★★ T4 · Treatment Rec. ★★★ T5 · Multimodal EEG ★★★★★
Novelty ★★★★★

T1 — New-Paper QA · 最新文献问答

~800–1200 QA对 问答 QA
Task Description

测试 LLM 对 2023–2025 年新发表癫痫文献知识的掌握与推理能力。分三个子任务：

  • ① Closed-book QA：直接测 LLM 内部知识时效性
  • ② RAG QA：给定检索段落后回答
  • ③ Evidence Grounding：回答中标注引文来源
Data Source
  • 主库：18,213篇癫痫论文中筛选 2023–2025 年子集 ~3,000篇
  • GPT-4o pipeline 自动生成 QA → 专家抽样验证 ~200条
  • 按发表时间分层：pre-cutoff / post-cutoff 对比分析
Evaluation Metrics
M1
Exact Match (EM) & Token F1 — 标准化QA评测
M2
BERTScore (F1) — 语义相似度
M3
Temporal Accuracy — 按论文发表年份分层报告（时效性核心指标）
M4
[RAG] Faithfulness Score — 答案与检索段落一致性
M5
[RAG] Citation Precision/Recall — 引文准确率
M6
Hallucination Rate — 医生评审（安全关键）
Baselines — C1 Closed-Source LLM
ModelProvider类型选择理由
GPT-4o (2024-11)OpenAI闭源SOTA最强通用LLM，知识截止2024-04，提供时效性对照
Claude 3.5 SonnetAnthropic闭源SOTA强推理，知识截止2024-04
Gemini 1.5 ProGoogle闭源SOTA长上下文支持，适合长文献段落
o1-miniOpenAI推理增强CoT推理增强版，对复杂推理QA有优势
Baselines — C2 Open-Source LLM
ModelProvider类型选择理由
Llama-3.1-70B-InstructMeta开源SOTA目前最强开源通用模型
Meditron-70BEPFL医学专用在医学文献上持续预训练
BioMedLM-2.7BStanford/NVIDIA生物医学小模型基线，PubMed预训练
Baselines — C3 SOTA Paper Methods
Method / PaperTask类型Reviewer视角：为什么选
MedRAG (Xiong et al., ACL 2024)医学RAG框架MIRAGE benchmark上的SOTA RAG系统，GPT-3.5+RAG达GPT-4水平，是最权威的医学RAG基线
Self-RAG (Asai et al., ICLR 2024)自适应RAG选择性检索策略，减少噪音注入，是RAG方向的ICLR best paper
MKRAG (AMIA 2025)医学知识RAG医学领域RAG专用，在MedQA-SMILE上+4%提升
PubMedQA baseline生物医学QA标准医学QA基线，提供可比的数字参考
🌟 Our Advantage vs. Existing Work：首个针对时效性（temporal recency）的癫痫QA benchmark；现有医学QA均基于静态知识库，无法测新发表知识的追踪能力。通过 pre-cutoff vs. post-cutoff 分层分析，揭示LLM知识截止对临床决策的影响。
Novelty ★★★★★

T2 — Patient Explanation Generation · 病历→患者可读解释

~300–500 对 生成 Generation
Task Description

将专业医学报告转换为患者可理解的通俗解释。目标读者：中学教育水平患者及家属（6年级阅读水平）。

子集：① EEG报告解释 ② MRI报告解释 ③ 出院小结患者版 ④ 用药说明简化

Data Source
  • 真实癫痫 EEG/MRI 报告（合作医院）
  • 神经科医生撰写标准患者版本作为 Gold Standard
  • 参考：MIMIC-III/IV discharge notes 数据构建方法
  • 对标：EHRNoteQA (NeurIPS 2024) 的评测框架
Evaluation Metrics
M1
ROUGE-L — 与金标准词汇重叠
M2
BERTScore — 语义相似度
M3
Flesch-Kincaid Grade Level — 可读性（目标 ≤ 8）
M4
Clinical Safety Score — 医生评审误导性信息率（安全关键）
M5
Information Completeness Rate — 关键医学事实覆盖率
M6
[Human] 患者理解度评分 — Likert 1–5，患者/家属盲评
Baselines — C3 SOTA Paper Methods（核心）
PaperTask类型Reviewer视角
Rust et al., Commun Med 2025出院摘要简化GPT-4o简化心脏科出院摘要，可读性显著提升；方法直接可移植到癫痫任务
Williams et al., JAMA Internal Med 2024LLM vs 医生摘要100例对比研究，提供评测框架参考；LLM质量与医生相当但错误更多
MEDISCHARGE (ACL BioNLP 2024)出院摘要生成Meditron-7B微调系统，提供开源baseline方法和代码
Koohi et al., JMIR Med Inform 2025EHR简化highlight辅助的LLM摘要，MIMIC-III数据集，完整性+正确性提升
EHRNoteQA (NeurIPS 2024 D&B)EHR QA Benchmark962个QA对评测27个LLM，Spearman 0.78与临床医生一致性，是最权威的EHR评测框架
🌟 Our Advantage：首个专注"医→患翻译"质量+安全性的癫痫生成benchmark；现有工作做出院摘要生成（医→医），未做患者版（医→患）；癫痫领域完全空白。同时引入Clinical Safety Score作为必要安全维度。
Novelty ★★★★

T3 — Variant → Personalized ASM · 基因变异→个性化用药

~400–600 病例vignette 推理 Reasoning
Task Description

给定癫痫患者的基因变异报告（基因名、变异位点、ACMG致病性分类）和发作表型，要求LLM推荐个性化ASM并给出推理链，避免禁忌药物。

覆盖基因：SCN1A, SCN2A, SCN8A, KCNQ2, CDKL5, STXBP1, TSC1/2, DEPDC5 等 20+ 基因

Output
  • ① 推荐用药（排序列表）+ 推荐理由
  • ② 禁忌药物列表 + 禁忌原因（机制说明）
  • ③ Chain-of-Thought 推理链（基因机制→药物靶点→推荐逻辑）

示例：SCN1A功能缺失→Nav1.1降低→钠通道阻断剂禁忌→推荐丙戊酸钠/氯巴占

Evaluation Metrics
M1
Drug Recommendation Accuracy — Top-1 / Top-3（vs 专家共识）
M2
Contraindication Avoidance Rate — 禁忌药物回避率（安全关键指标，越高越好）
M3
Chain-of-Thought Quality Score — 推理链包含正确机制，医学专家评分1-5
M4
Rare Gene Performance — 常见 vs 罕见基因准确率差异
M5
VUS Handling Score — 变异不确定性情境下的处理合理性
Baselines — C3 SOTA Paper Methods（核心）
Paper类型Reviewer视角
Lamberink et al., JAMA Neurol 2022
Transformer-based ASM response predictor		治疗预测1798例多中心队列，AUROC 0.65，目前最好的ML治疗预测基线，直接可比
Lamnisos et al., Epilepsy Res 2024
ML ASM prediction systematic review		系统综述37项ML研究，提供tree-based/SVM的性能上界参考
JSON Epilepsy Matcher
(Tombini et al., J Clin Neurosci 2024)		GPT+结构化症状+EEG准确率93.8%，含用药推荐部分，最接近本任务的现有工作
ClinVar/ACMG automated classifiers
(SIFT, PolyPhen-2, CADD, REVEL)		变异分类传统变异致病性预测工具，作为基因解读基线；reviewer会认可这些标准参照
Diessen et al., Epilepsia 2024
LLM epilepsy care review		LLM综述系统评估LLM在癫痫中的优缺点，记录了LLM治疗推荐错误案例（如丙戊酸妊娠错误）
🌟 Our Advantage：首个将基因变异解读与个性化ASM推荐打通的LLM推理benchmark；现有工作要么做变异分类（ML），要么做治疗预测（ML），两者结合+LLM推理链几乎空白。
Novelty ★★★

T4 — Treatment Recommendation · 抗癫痫药物治疗推荐

~600–800 问题 推荐 Recommendation
Task Description

给定患者临床信息（发作类型、年龄、性别、共病、妊娠状态、过敏史），推荐首选及备选ASM，并解释选择理由。

子集：① 普通成人 ② 儿童 ③ 育龄女性（妊娠相关）④ 难治性癫痫（≥2药失败）

Data Source
  • AES 2024 ASM Summary（4th Edition）
  • ILAE Treatment Guidelines (2023/2024)
  • AAN/NICE Epilepsy Guidelines
  • 格式：多选题 + 开放式题，各半
Evaluation Metrics
M1
Multiple-choice Accuracy — 标准化格式，自动评测
M2
Top-3 Accuracy — 前3个推荐中包含正确答案
M3
Guideline Consistency Rate — 是否符合当前ILAE/AES指南
M4
Safety Score — 禁忌药物推荐率（越低越好）
M5
Expert Agreement — 开放式题，专家Likert评分1-5
M6
子集性能对比 — 普通/儿童/育龄女性/难治性各自准确率
Baselines — C3 SOTA Paper Methods（核心）
Paper类型Reviewer视角

Novelty ★★★★★

T5 — Multimodal EEG + Clinical Text · 多模态诊断

~500–800 对 多模态 Multimodal
Task Description

将EEG信号的量化特征（文本化表示）与临床描述文本联合输入LLM，完成综合征诊断推理。

核心创新：EEG特征文本化——将信号特征（棘波频率/背景节律/放电模式）转化为结构化文本，绕开信号处理瓶颈，使LLM可直接处理。

Data Source
  • TUH EEG Corpus（Temple University Hospital，最大公开EEG数据集）
  • CHB-MIT Scalp EEG（MIT，儿童癫痫）
  • 自动化特征提取 pipeline（PyEDFlib）→ 文本化
  • 金标准：专科医生诊断标签（ILAE 2022综合征定义）

M1
Syndrome Classification Accuracy — 主要指标
M2
Macro-F1 — 处理类别不平衡
M3
Multimodal Gain Δ — vs EEG-only（单模态消融）
M4
Multimodal Gain Δ — vs text-only（单模态消融）
M5

M6
