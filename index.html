Two-Layer Task Framework

Challenge: The task selection lacked a principled framework — reviewers would ask "why these five tasks specifically?"

We adopt a two-layer concentric framework: the inner layer defines three core ML task types (Generation / Prediction / QA & Classification); the outer layer contains concrete clinical instantiations. Every task has an explicit category membership, and together they provide systematic coverage of the epilepsy clinical AI task space.

EpiGraph KG — Four Advisor Fixes

Challenge: The KG construction pipeline had four systematic flaws affecting reproducibility, source traceability, and the validity of downstream performance evaluation.

All four issues are addressed in new_test_v9.py (2,350 lines, syntax validated). Every change is designed to support ablation experiments that quantify each fix's contribution to downstream QA accuracy.

Fix 1 · Branch Factor
--max_branch Parameterization
Hardcoded at 8 — impossible to study the relationship between KG density and downstream performance
Parameterized (default 8). Ablation: K = 4 / 8 / 16 / 32, measuring triplet count, chain depth, and QA accuracy
Fix 2 · Source Provenance
canonical_source Field
Entities lacked source attribution — no way to distinguish UMLS-verified entities from free-text extractions
Every entity carries canonical_source (UMLS / MeSH / OMIM / CHEBI / HPO / ILAE_2022 / AES_2024 / curator). triplets.json includes head_source and tail_source per triplet
Fix 3 · LLM Sampling
Per-Paper Sampling Fix
Sampling was applied per-entity (logic error), causing uneven LLM call distribution across papers
Refactored into LLMExtractor class with extract_from_paper() method. 3-retry + exponential backoff. Ablation: ratio = 0 / 0.3 / 1.0
Fix 4 · Text Scope
--use_fulltext Flag
No control over input scope — abstract-only vs. full-text performance never compared
Added --use_fulltext (default: abstract + first 3 pages; full-text: 20 pages). Ablation: triplet recall and QA accuracy across both modes

T2 — Patient Explanation Generation

Challenge: Gold standard cannot be LLM-generated (circular evaluation problem), but publicly available epilepsy patient education datasets are virtually nonexistent.

Use PMC plain-language summaries as gold standard. A subset of PMC articles provides author-written lay summaries (not LLM-generated), which naturally form technical abstract ↔ patient-friendly pairs. Target: 500+ pairs, filtered by epilepsy keywords via the PMC API (free access).

T3 — Genetic Variant → ASM Recommendation

Challenge (advisor suggestion): How to construct a validation set? External genetic databases lack concrete clinical medication decision records tied to specific variants.

Build the validation set directly from the collected paper corpus. Extract clinical vignettes from case reports and cohort studies: genetic variant + patient phenotype (seizure type, age of onset, EEG features, medication history) → the treatment decision reported in the paper serves as gold standard. Data is fully traceable and gold standard has explicit literature backing.

⚠ Key risk: dataset may be too small. Run a 50-paper pilot extraction first to verify vignette yield before committing to full-scale processing.

T4 — Treatment Recommendation

Challenge: Treatment recommendation has no single correct answer — different clinicians may reasonably recommend different ASMs for the same patient. How do we define a gold standard?

Rather than requiring a unique correct answer, we evaluate against clinical guideline concordance (ILAE 2022 + AES 2024) and KG evidence coverage. This reflects clinical reality (multiple recommendations can be simultaneously valid) while providing objective, quantifiable evaluation dimensions.

T5 — EEG Report Generation Redesigned

Challenge (advisor suggestion): Original T5 was an EEG classification task with limited clinical value. How do we redesign it as a more meaningful generation task?

Redesigned as KG-assisted EEG report generation: given EEG description text → retrieve EEG-syndrome mappings from KG → LLM generates clinical impression. Writing the impression is where neurologists spend the most time, making this the highest-value clinical AI application point.

Two sub-tasks: T5a (pipeline) — text description → KG → impression (feasible now); T5b (end-to-end) — raw EEG signal → description → KG → impression (→ Future Work).

Unsupervised Detection — Feasibility Assessment

Challenge (advisor suggestion): Can we align KG entity embeddings with EEG signal embeddings to enable unsupervised seizure event detection using seizure EEG datasets?

The direction is technically novel. However, three critical blockers make it unsuitable as a primary task in the current timeline. Recommendation: treat as Future Work.

✓ Data is available

CHB-MIT (23 patients, labeled seizure events) and TUH EEG are both publicly accessible with complete seizure annotations.

⚠ EEG encoder quality is uncertain

BENDR and EEG-GPT are available but require fine-tuning; variable quality caps the alignment ceiling.

⚠ Alignment training data is scarce

Alignment requires paired (EEG segment, clinical description) training data — limited supply within TUH.

✗ Reviewer challenge unavoidable

Zero-shot detection will necessarily underperform supervised methods. Time cost ≥ 2–3 months. Risk-reward ratio is unfavorable for current project scope.

→ Recommendation: Future Work

Describe the KG-EEG alignment architecture in the paper. If more than 3 months remain before submission, a 50-clip preliminary experiment can serve as a proof of concept — not a primary contribution.
